내가 쓰는 vscode extension
Created 2020-04-30| Updated 2020-05-04|Vscode
Post View:

vscode,,

내가 18학번이니까 2년 전인가 visual studio 깔려다가 실수로 깔아서 그냥 써봤는데 확장(extesion) 기능이 있는게 좋아서 쭉 쓰게 됬다. 초창기엔 json 막 설정하고[^1] 오류 팡팡 터지고 컴파일도 안되고해서 이거 굳이 써야하나 했는데 최근 들어서 괜찮다란 생각이 자주든다. 개발자들 사이에서 인기도 점점 많아지고 있지 않는가

[^1]: 아무것도 몰라서 고생만 신명나게 했다;;

Extension 소개

Markdown All in one

  • 장점

    • 역시나 귀찮음 해결
      • list 작성 때 ENTER 누르면 자동으로 * 을 만들어줌
      • TAB누르면 * 한 칸 옆으로 감
    • 생산성이 확실히 올라간다.
      • 막 파일 찾아서 넣어야하는데 파일 경로 자동으로 표시되는등

  • 근데 아직 footnote(주석) 기능은 없는 듯

  • TIL이나 포스트를 MD 파일로 작성하는데 쓸데 없이 시간 낭비가 줄었다.

Markdown Foootes

  • 위에 말했던 각주를 달아주는 EXTENSION
  • MARKDOWN 특성상 띄어쓰기 잘못하면 인식 안될수도 있는데 잘 인식함
  • 편의상 쓰는 글 아래 각주 내용을 두는게 깔꼼하다
  • 확실히 각주 달아야 가독성이 올라가서 쓰게 되는거 같음

Auto Open Markdown

  • 내가 쓴 MD 파일 PREVIEW를 띄어주는 EXTENSION이다
  • 가끔 안뜨긴 하는데 파일 여는 순간 바로 옆에 뜬다
  • 가끔 오타 나서 문법 적용안되는거 미리 볼 수 있어서 편함
  • PREVIW 반응도 빠른 편

GitLens

  • VSCODE 자체 GIT이 있어도 이거 쓰는 이유를 꼽으라면
    • CODE를 누가 짰는지 확인할 수 있음
    • CODE 마지막 수정 시간도 뜸
    • 코드에 마우스 가져가면 어떤 커밋인지도 보여줌[^2]
  • 기능이 더 많긴한데 일단은 이정도만 해도 만족

[^2]: 이정도만 해줘도 박수 쳐야지

Andent-rainbow

  • TAB이나 SPACE 바 사용정도? 공백을 색깔로 표현해줌
  • 가시적으로 보이니까 의식적으로 코드를 깔꼼하게 적는 듯
