그냥 내가 아는 것, 배운 것 정리?

뭐랄까 나름 공부하다보니 생기는 것들 알게 되는 것들 적으려고 함 써본다. 예를 들어 문제 발생시 접근 방식 의심하는 것, 공부하고 싶은 것들은 주루룩 적는거지

I learned

GAIN

  • 민감도
  • INIPUT SOURCE 받을 때 1차적으로 거친다
  • FADER를 0으로 둔 후 적정 레벨까지 받자
    • 너무 많이 받을 경우 소스가 깨질 수 있다.

EQ

  • GAIN 그리고 디지털의 경우 이펙터를 거쳐 들어오는 소스를 만질 수 있다.
  • 활용
    • 하울링 발생 시 하울링 발생 주파수를 깎아 하울링을 없앨 수 있음
    • 악기나 사람의 음색? 톤을 만질 수 있음
    • 그래도 BOOST는 조심하자 필요에 따라 만질 수 있지만
    • 보통은 깎는 것을 시작으로 배운다.
  • 오개념
    • 싱어의 저음이 부족하다고 LF를 올린다고 문제가 해결되지 않는다.
      • 그쪽은 사람 저음 영역이 아니여
    • 반대로 싱어의 목소리를 밝게 하고 싶다고 HF를 무조건 올리지말자
      • 오히려 치찰음이라던가 귀가 아플 수 있음
    • PEAK 뜨는 건 음색이 아니다
      • SOURCE가 과해서 깨지고 있는 거임
    • RTA 볼 때 피크는 아니지만 올라오는 것들이 하울링의 불안요소는 아님
  • 무지향성 마이크로 RTA 보면서 EQ 깎으면 깔꼼하더라
  • 기계가 있지만 상황 바이 상황이 있으니 귀도 열려있어야함

COMP

  • SOURCE의 다이내믹 레인지를 줄여줌
  • RATIO : 비율
  • 활용
    • SOURCE가 깨질 경우
    • 악기 톤
    • 세션과 악기의 존재감 조절
  • 악기의 경우 걸치는 구간에 사용

ROUTING

  • 공유기
    • X32의 경우, 콘솔에 공유기를 연결하여 패드라던가 다른 기기에서 접근 가능
    • IP 입력해야함
      • 공유기 회사마다 초기 설정 IP가 있으니 들어가서 확인 후 세팅
    • 뒤에 랜선을 꽂을 때 WAN아니라 WLAN에 꽂아야함
      • 이거 이유 알고 있었는데 까먹었다.
  • USB
    • USB와 PC를 연결해 멀티트랙 레코딩 할 수 있음
      • 근데 드라이버 오류가 있어서 인식이 잘안됨(WINDOW의 경우)
    • 베링거 공홈에 들어가서 FIRMWARE 받아야함(WINDOW의 경우)
  • OUT
    • Network랑 똑같음
    • INPUT/OUTPUT 확인해야함
      • 모든 음향 장치 셋업에 있어 공통 사항임
      • SIGNAL FLOW가 제대로 되어 있는가
    • 스테이지 박스가 있을 경우 랜선으로 빠지니까 확인해야함
      • 스테이지 박스는 AESOUT임
      • X32의 경우 ROUTING 들어가서 Local이 아니라 AES out으로 빼야함
    • X32가 4.0 펌웨어 업데이트 되면서 USER 아웃이 생겨서 커스텀 가능

      Effector

사실 EQ,GATE,COMP 모두 이펙터이다 여기서 이야기하고 싶은 건 그 외에 것들이다. 난 주로 X32 가끔 XR18 쓰니까 다른 믹서는 모르겠는데 디지털 믹서에서는 기본으로 EQ,GATE,COMP 정도는 있어서 나머지 것들만 따로 뺐다.

종류

  • Dessor
    • 치찰음 줄이는 용도
      • 근데 별로 안씀 그냥 EQ 잡는게 더 깔끔한 것 같음
      • 과하면 답답한 소리남
  • PLATE EFFECTOR
    • 철판에 소리를 보내고 반사되는 소리
    • 무난하게 싱어나 세션들 리버브로 쓴다
  • TAP TEMPO
    • 이거 재밌는게 박자 맞춰서 누르면 딜레이 조절 할 수 있다.
  • 이펙터 조절
    • 이펙터가 있어도 원 SOURCE가 중요한건 팩트
    • 딜레이가 심할 경우 소리가 돈다.

세부 세팅?

  • Decay

    스피커

  • 모니터 스피커의 경우 PREFADER로 조절해야함
  • 스피커 각도
  • 스피커 출력 한계 꼭 확인해야한다.
    • 잘못하면 팡

PHANTOM

  • 악기나 콘덴서 마이크에 전기를 넣는데 사용
  • 뮤트 안시키고 XLR 라인 뽑으면 이제 팡 소리 나는 거지
    • 마이크는 괜찮은데 스피커가 고장 날 수 있음
    • 갑분싸 만드는거지
  • 다이나믹 마이크에 팬텀을 넣는다고 뭔일이 나진 않지만 그래두 빼두자
  • 48V 전기이다.

문제 접근

하울링 생겼다?

[^1]: 페이더 내려두 되긴하는데 모르겠다 하면 언능 내려야지 그래도 메인 페이더 내리는 건 한 번 생각해봐야한다.

  1. 하울링 생기는 소스를 찾아 뮤트[^1]
    1. 마이크
    2. 어쿠스틱 기타
      • 어쿠 GAIN이 높을 경우 소리가 들어가서 생길 수 있음
  2. 원인 파악
    1. 하울링 발생 소스의 위치
      • 싱어가 메인 스피커 앞에서 노래 부를 경우 자리를 옮겨달라하자
      • 어쿠에서 발생할 경우 게인을 낮추자
      • 모니터 스피커 때문에도 생기는데 이때 페이더를 낮추던가 문제가 되는 소스를 줄인다
    2. 공간의 문제인가?
      1. 홀 튜닝
        • 우리나라처럼 4계절이 있는 경우 더 그렇긴 하는데 계절마다 습도,온도 등이 달라지니 계절이 바뀔 때 마다 해주자
      2. 하울링뜨는 주파수 파악
        • 해당 주파수를 깎아 버린다.
          • 해당 주파수를 안 받아버리니 하울링 해결!

소리가 안나요

  • Signal flow를 확인해야한다.

  1. SIGNAL이 들어온다

    1. Mute 된 것이 있는가 확인한다(DCA,MATRIX,MUTEGROOP 등 아차 하는 것들)
    2. ROUTING 확인
      • ROUTING SETTING이 잘못되었는가
        • BUS를 잘못 보냈다
        • X32의 경우 STERO BUS로 보내고 있는지 확인
        1. 소리가 안나는 곳으로 간다.
      • 소리가 나오는데 작아서 못드는 건가
      • 보내주는 SOURCE가 너무 적은건 아닌가
      • 인이어 세팅이 잘되어 있는가
      • 장비 고장 의심
        • 채널을 옮겨본다
        • 라인을 바꿔본다

  2. SIGNAL이 안들어온다

    • 세션이나 콘덴서 마이크 의 경우 PHANTOM 확인
    • 올바른 채널에 꽂혀 있나
