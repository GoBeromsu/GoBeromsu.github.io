Git을 왜 쓰게 되었는가?

사실 github desktop 버전을 사용해왔는데 확실히 편하기도 하고 나름 merge라던가 pull request도 편하게 할 수 있어 사용했는데 나름 한계가 느껴져서 jekyll 블로그를 만들면서 이참에 그냥 git을 썼다. 귀찮음이 컸던거 같다. GUI 냅두고 굳이 Command를 쳐가면서 해야되냐 이런거였지만 막상 쓰니까 적응되서 좋은거 같다. git같은 협업 툴을 쓸 줄 알아야한다 생각해서 자주 써보도록 해야겠다.

git repo 구조

  • Working dir - index(stage) - Head <==> Remote repo

github desktop이랑 차이가 여기인거 같다. github desktop은 working dir에서 변화가 생기면 바로 감지하고 commit할 수 있다. 중간 과정이 자동으로 이루어지는 건데 장단점이라면 편하다랑 편한만큼 git 구조를 모르게 된다.

git config ~

git init

  • local에서 git 저장소를 만드는 명령어이다
  • project dir로 이동해서 git init 명령어를 실행한다.

git clone

  • 다른 프로젝트를 복사 해 올 수도 있다.

git status

  • 현재 브랜치를 확인할 수 있다.

git add

  • working dir의 작업물을 index로 보내는 명령어
  • git add . : 할 경우 untracked files를 몯 한번에 추가 할 수 있다.

git commit

  • index에 stage 된 파일들을 commit 하는 명령어
  • git commit -a : staging(add) 생략하고 바로 커밋까지 할 수 있다.
