서버리스란 무엇인가?

  • 개발자가 직접 서버를 관리하지 않아도 된다
    • 백엔드가 없다는 뜻은 아님
  • 백엔드를 작은 함수단으로 쪼개 서버에 올림
  • Request가 들어왔을 때만 서버가 일함
    • 그래서 내가 수행한 함수만큼 돈을 내는 거야

서버리스의 단점

위에서는 장점을 이야기 했으니 단점으로 가보자공

  1. 아무래도 Request가 있을 때만 실행하다보니 잠든 함수를 깨우는 시간이 걸림
  2. 서버 제공자에게 의존하게됨
  3. 서버리스가 서버를 옮기는건 쉽지 않음
    1. 어플리케이션 구조를 바꿔야하거든
  4. 서버에 대한 통제권을 잃을 수 있음

누가 써야하는가?

  • 사이드 프로젝트를 하는 친구들
  • 빠른 시제품을 내고 싶을 때
    • 코드에만 집중하면 되기 때문
  • 서버 관리하고 설정하는데 시간을 아끼고 싶은 경우
